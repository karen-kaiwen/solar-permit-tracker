# 科技產業園區太陽光電申設案件清冊自動化

每月自動比對約 9 萬筆太陽光電申設資料，篩選出當月新增的同意備案、
設備登記及併網案件，取代人工 Excel 篩選作業，將每月例行作業時間縮短 50–80%。

## 專案概述

每月需向長官呈報當月新增的同意備案及設備登記案件。
原本作法是直接在 Excel 中人工篩選兩份來源總表，但合計約 9 萬筆資料、
欄位繁多，頻繁出現 Excel 閃退、需來回核對的狀況，容易抓錯資料。

本腳本以 R 撰寫，每月只需更新兩個變數（檔案路徑、報告日期），
即可自動完成資料清理、合併、比對，並匯出僅含必要欄位的 Excel 報表，
供製作呈報表單使用。

## 技術挑戰與解決方案

### Excel 序號日期轉換
- **問題**：來源檔案中部分日期欄位以 Excel 序號格式儲存（如 `45291`），
  直接讀取會得到數字而非日期，無法進行年份篩選或日期運算。
- **解決方案**：統一使用 `as.Date(as.numeric(...), origin = "1899-12-30")`
  將序號還原為正確的 Date 物件，確保日期比對結果正確。

### 重複篩選邏輯維護性
- **問題**：同意備案與設備登記兩份資料需套用相同的六項篩選條件
  （聯絡人、年份、案件屬性、設置位置、設備型別、發電設備種類），
  若條件異動需同時修改兩處，容易遺漏。
- **解決方案**：將共用篩選邏輯抽成 `filter_solar_cases(df, date_col)` 函數，
  條件異動只需修改一處，兩份資料自動同步套用。

### 欄位名稱空白造成比對失敗
- **問題**：上個月清冊的欄位名稱含有不可見空白字元，導致 `anti_join`
  無法正確比對欄位，產生錯誤的新增結果。
- **解決方案**：讀入後立即以 `rename_with(~ gsub("\\s+", "", .x))`
  移除所有欄位名稱中的空白，確保比對基準一致。

### 來源資料重複編號導致合併結果膨脹
- **問題**：來源總表為外部系統匯出，格式不在本腳本控制範圍內。
  若 `設備登記總表` 中同一個 `同意備案編號` 對應多筆記錄，
  `left_join` 後該筆同備會被展開成多列，造成清冊筆數錯誤膨脹，
  且不會有任何錯誤訊息提示。
- **解決方案**：在合併前以 `stopifnot` 對兩份清理後的資料進行重複編號檢查，
  一旦發現重複即中斷執行，避免產出錯誤報表。
- **資料品質檢查範例**:
  ```
  stopifnot("發現重複的同意備案編號！" = !any(duplicated(同意備案總表_f$同意備案編號)))
  ```

## 功能總覽

| 功能 | 說明 |
|------|------|
| 🔍 **共用篩選函數** | 將六項業務篩選條件封裝為 `filter_solar_cases()`，兩份總表共用 |
| 📅 **日期格式統一** | 自動處理 Excel 序號與字串兩種日期格式，確保比對正確 |
| ✅ **合併前資料驗證** | 以 `stopifnot` 檢查重複編號，發現異常立即中斷，避免產出錯誤報表 |
| 🔗 **跨表合併** | 以同意備案編號為鍵，left join 同備與設登資料 |
| 📊 **三類新增比對** | 分別比對新增同備、新增設登、新增併網三種情境 |
| 📤 **自動命名匯出** | 輸出檔名自動帶入報告日期，避免覆蓋歷史檔案 |

## 使用前置條件

- R 4.0 以上版本
- 已安裝以下套件：

```r
install.packages(c("tidyverse", "lubridate", "readxl", "writexl", "openxlsx"))
```

- 三份來源檔案：
  - 同意備案總表（`.xlsx`）
  - 設備登記總表（`.xlsx`）
  - 上個月清冊（`.xlsx`，第一列為說明列，skip = 1）

---

## 設定與執行

1. 開啟腳本，修改最上方的路徑設定區塊：

```r
PATH_同意備案   <- "YOUR/PATH/approve_filling_YYYYMMDD.xlsx"
PATH_設備登記   <- "YOUR/PATH/設備登記總表_YYYYMMDD.xlsx"
PATH_上個月清冊  <- "YOUR/PATH/上個月清冊.xlsx"

REPORT_DATE    <- "YYYMMDD"   # 本次報告日期，格式：民國年月日
```

2. 執行全部程式碼，若來源資料有重複編號，程式將中斷並顯示提示訊息。
3. 確認無異常後，輸出檔案將存至工作目錄。

## 輸出檔案說明

| 檔案名稱 | 內容 |
|---------|------|
| `同備設登科技產業園區...reference_{DATE}.xlsx` | 同備與設登合併後的完整清冊 |
| `新增同備_{DATE}.xlsx` | 本月新增、上個月清冊中尚未出現的同意備案 |
| `新增設登_{DATE}.xlsx` | 本月新增、上個月清冊中尚未出現的設備登記 |
| `新增併網_{DATE}.xlsx` | 同備已存在但設登為新增或空白的併網案件 |

## 已知限制

- 日期格式依賴來源檔案一致性；若來源格式異動，需同步調整 `mutate` 中的轉換方式。
- 路徑設定為本機絕對路徑，換機使用時需手動更新。
- 腳本不驗證來源欄位是否存在，若來源總表欄位名稱異動，需同步更新 `select` 與 `rename`。
